# 辐照监测可视化与智能化分析系统设计

## 1. 文档目的

本文档定义 `irradiation_visualizatio_analysis` 从原有“辐射超标地面点位与手套孔研判平台”派生为“辐照监测可视化与智能化分析系统”的目标、数据规范、空间布局、系统架构、页面结构、分析规则、预测方法、Excel 生成与导出方案以及验收要求。

本次改造采用“分层重构并逐步替换”路线：保留 Streamlit 技术栈和现有界面的成熟交互思路，但替换旧业务模型，并将 Excel、状态判定、布局、分析、预测和报告逻辑从大型 `app.py` 中拆分。

## 2. 背景与目标

### 2.1 系统背景

系统面向多个房间及房间内设备产生的长期辐照监测数据，提供空间可视化、阈值判定、历史趋势、异常事件、风险排序、简单预测和 Excel 分析报告。

### 2.2 核心目标

1. 管理 20 个房间、每房 10 台设备，共 200 台设备。
2. 支持每台设备通常 1 至 2 种辐照监测类型，并允许监测类型名称动态扩展。
3. 从单个长期总表或多份日表、阶段表中读取 Excel 长表数据。
4. 根据每条记录自带的预警值和控制标准进行三级状态判定。
5. 用房间和设备布局替换旧系统的 840 点网格、地面点位、手套孔和障碍物模型。
6. 提供设备、房间和系统整体层面的趋势、异常、风险与预测分析。
7. 提供正式录入模板、模拟数据生成器和多工作表分析报告。

### 2.3 非目标

第一版不包含：

- 在线传感器或实时消息队列接入。
- 用户权限、审批流和数据库持久化。
- 人工覆盖阈值的编辑界面。代码结构需为后续功能保留扩展点。
- 将预测结果作为法定监测结论或事故处置依据。
- 复杂机器学习或不可解释的黑盒预测模型。

## 3. 术语与标识

### 3.1 房间与设备 ID

- 房间 ID：`R01` 至 `R20`。
- 设备 ID：`R01-D01` 至 `R20-D10`。
- 设备 ID 的房间前缀必须与同一行的房间 ID 一致。
- 系统内部可生成 1 至 200 的连续序号用于排序，但界面、Excel 和导出均以结构化 ID 为主。

### 3.2 监测类型

监测类型由 Excel 自由填写，例如“γ剂量率”“中子剂量率”。系统不内置固定类型清单，而是根据有效数据动态生成筛选项、图例和统计分组。

业务上每台设备通常有 1 至 2 种监测类型；数据内核不硬性限制为两种，以便后续扩展。

### 3.3 状态等级

每条监测记录按以下边界判定：

| 条件 | 状态 | 业务含义 |
|---|---|---|
| `监测值 < 预警值` | 正常 | 未达到干预阈值 |
| `预警值 <= 监测值 < 控制标准` | 预警 | 请求人为干预 |
| `监测值 >= 控制标准` | 事故级 | 必须开展隐患排查和事故调查 |

同一设备存在多个监测类型时，设备状态取所有当前有效监测类型中的最严重状态。房间状态取房间内所有设备的最严重状态。

## 4. 空间布局

### 4.1 房间总体布局

房间按 5 列、4 排排列：

```text
R01  R02  R03  R04  R05
========== 主过道 A ==========
R06  R07  R08  R09  R10
R11  R12  R13  R14  R15
========== 主过道 B ==========
R16  R17  R18  R19  R20
```

入口规则：

- 第一排房间入口位于房间下边界中央，朝向主过道 A。
- 第二排房间入口位于房间上边界中央，朝向主过道 A。
- 第三排房间入口位于房间下边界中央，朝向主过道 B。
- 第四排房间入口位于房间上边界中央，朝向主过道 B。
- 图中入口统一标注为“入口”。

### 4.2 三种设备布局

#### A 型：平行双排

- 远离入口的一排放置 6 台设备，按六等分中心均匀排列。
- 靠近入口的一排放置 4 台设备，在入口左右对称分布。
- 入口位于四设备一排的中央。
- 入口通过支路连接两排之间的横向操作通道。
- 入口轴线和操作通道不得被设备图标遮挡。

#### B 型：双侧尽头

- 左侧 4 台、右侧 4 台，沿中央操作通道等间距排列。
- 远离入口的操作通道尽头横向放置 2 台设备。
- 入口位于面向主过道一侧的房间边界中央，并与中央操作通道对齐。

#### C 型：环形分布

- 10 台设备沿房间四周分布。
- 房间中央保留环形操作通道。
- 入口位于面向主过道一侧的房间边界中央，通过径向通道接入环形通道。
- “环形通道”标签放在中央净空区域，不与设备 ID 交叠。

### 4.3 房间布局类型分配

三种布局按房间顺序循环：

| 布局 | 房间 |
|---|---|
| A 型 | R01、R04、R07、R10、R13、R16、R19 |
| B 型 | R02、R05、R08、R11、R14、R17、R20 |
| C 型 | R03、R06、R09、R12、R15、R18 |

### 4.4 设备状态着色

布局图至少使用以下状态颜色：

- 正常：绿色。
- 预警：橙色或黄色。
- 事故级：红色。
- 无有效数据：灰色。
- 当前选中：使用独立描边或高亮，不覆盖状态颜色。

## 5. 输入 Excel 规范

### 5.1 数据结构

输入采用长表结构，每行表示“一个时间、一个设备、一种监测类型”的一次监测记录。

系统必须同时支持：

- 单个 Excel 中包含长期历史记录。
- 批量上传多个日表或阶段表并自动合并。

支持 `.xlsx`；第一版不要求兼容旧版 `.xls`。

### 5.2 必填列

| 列名 | 类型 | 说明 |
|---|---|---|
| 监测时间 | 日期或日期时间 | 允许仅日期或精确到时间 |
| 房间ID | 文本 | `R01` 至 `R20` |
| 设备ID | 文本 | `Rxx-D01` 至 `Rxx-D10` |
| 监测类型 | 文本 | 动态类型名称 |
| 监测值 | 数值 | 当前监测结果 |
| 单位 | 文本 | 当前监测类型的计量单位 |
| 预警值 | 数值 | 人为干预阈值 |
| 控制标准 | 数值 | 事故级法定控制标准 |

### 5.3 可选列

- 房间名称
- 设备名称
- 数据来源
- 备注

### 5.4 工作表识别

- 生成器默认使用工作表名称 `监测数据`。
- 导入器不强制工作表名称；它扫描工作簿并选择包含全部必填列的工作表。
- 一个工作簿存在多个符合条件的工作表时，系统要求用户选择或明确显示采用的工作表。
- 第一版不要求独立的“设备阈值配置”工作表；预警值与控制标准在每条监测记录中重复保存。

### 5.5 时间精度

- 日期时间按原值保存。
- 仅日期的记录在内部归一化为当天 `00:00:00`，同时保存 `date_only` 原始精度标记。
- 界面不得把仅日期记录描述为真实零点采样。

### 5.6 阈值与单位

- 每条记录使用自身携带的预警值和控制标准进行历史状态判定。
- 预测结果使用该设备、该监测类型最近一条有效记录的阈值。
- 必须满足 `预警值 <= 控制标准`，否则该行是阻断错误。
- 同一设备和监测类型允许阈值随时间变化。
- 同一设备和监测类型出现不同单位时，系统发出警告，并按单位拆分趋势序列，禁止直接混合计算。

## 6. 导入、合并与数据质量

### 6.1 标准处理流程

```text
上传文件
  -> 识别工作表和字段
  -> 解析与标准化
  -> 业务校验
  -> 合并和去重
  -> 生成统一监测记录集
  -> 状态、分析和预测
```

### 6.2 记录主键

重复与冲突主键为：

```text
监测时间 + 房间ID + 设备ID + 监测类型
```

单位不是主键的一部分，但单位变化会产生数据质量警告并拆分趋势序列。

### 6.3 重复与冲突

- 主键及其余字段完全相同：自动去重。
- 主键相同但监测值、单位、阈值或其他业务字段不同：标记为冲突。
- 默认分析采用最后导入的一条冲突记录。
- 所有冲突版本必须保留在审计结果中，并导出到“数据质量问题”工作表。
- 文件上传顺序必须在界面中可见，以解释“最后导入”的含义。

### 6.4 错误等级

#### 阻断错误

以下记录不参与分析：

- 缺少必填列或必填值。
- 房间 ID、设备 ID 不合法，或设备与房间不匹配。
- 监测值、预警值或控制标准无法解析为有限数值。
- `预警值 > 控制标准`。
- 监测时间无法解析。

#### 可继续警告

- 房间名称或设备名称缺失。
- 日期只有天级精度。
- 同一主键存在冲突。
- 同一设备、监测类型出现单位变化。
- 同一设备、监测类型出现阈值变化。
- 部分设备或房间在所选时间范围内没有数据。

### 6.5 数据质量摘要

页面和分析报告必须包含：

- 上传文件数。
- 原始行数。
- 有效行数。
- 阻断错误行数。
- 自动去重行数。
- 冲突主键数。
- 时间覆盖范围。
- 有数据的房间和设备数量。
- 设备覆盖率。
- 监测类型列表。

## 7. 时间快照与聚合规则

### 7.1 默认快照

导入完成后，默认快照时间为数据集中的最大监测时间。

### 7.2 选定时间点的设备状态

当用户选择一个时间点时：

1. 对每个“设备 + 监测类型 + 单位”序列，取不晚于选定时间的最近一条有效记录。
2. 将这些记录逐条判定状态。
3. 设备状态取最严重状态。
4. 房间状态取房间内设备最严重状态。
5. 同时显示记录时间和距选定时间的时长，避免把过旧数据误认为实时状态。

若某设备在选定时间之前无有效记录，则显示“无有效数据”。

### 7.3 时间范围分析

时间范围筛选用于趋势、异常事件、排名和预测。空间总览可在时间点模式与时间范围汇总模式之间切换：

- 时间点模式：按最近有效记录展示状态。
- 时间范围模式：按范围内最严重状态展示，并在详情中显示最新状态和最严重事件时间。

## 8. 系统架构

### 8.1 技术路线

- 继续使用 Streamlit 作为应用框架。
- `app.py` 缩减为应用入口和页面装配。
- 使用 pandas 处理表格和时间序列，使用 openpyxl 读取、生成和格式化 Excel。
- 继续使用 Plotly 生成交互式空间图和趋势图。

### 8.2 模块边界

建议代码结构：

```text
app.py
src/
  irradiation_analysis/
    models.py
    excel_io.py
    validation.py
    status.py
    layout.py
    analytics.py
    forecast.py
    reporting.py
    generator.py
    ui/
      import_page.py
      overview_page.py
      trends_page.py
      intelligence_page.py
      reports_page.py
tests/
```

职责：

- `models`：房间、设备、监测记录、质量问题、异常事件和预测结果。
- `excel_io`：工作簿读取、工作表识别、字段映射和标准化。
- `validation`：字段、ID、数值、时间、单位和阈值校验。
- `status`：记录、设备和房间状态计算。
- `layout`：20 个房间、三类设备布局和 Plotly 坐标。
- `analytics`：趋势、异常事件、接近阈值、快速增长和风险排名。
- `forecast`：设备监测值和整体风险数量预测。
- `reporting`：多工作表分析报告。
- `generator`：录入模板与模拟数据。
- `ui`：Streamlit 页面与会话状态，不承载核心业务算法。

## 10. 智能分析

### 10.1 异常事件

对每个“设备 + 监测类型 + 单位”序列：

- 连续的预警或事故级记录合并为一个事件。
- 当状态恢复正常时结束事件。
- 事件至少包含开始时间、结束时间、持续时间、最高状态、峰值、峰值时间和阈值。
- 仅日期数据的持续时间以自然日表达。

### 10.2 快速增长

快速增长检测使用近期趋势与历史波动结合：

- 至少需要 3 个有效时间点。
- 按真实时间间隔计算近期线性斜率。
- 当斜率为正，且近期增量显著高于该序列历史典型波动时标记。
- 界面显示触发原因、使用窗口和斜率，不只输出布尔结论。

### 10.3 接近预警值

满足以下条件之一可列为接近预警：

- 当前值达到预警值的 80% 以上但尚未进入预警。
- 当前值虽低于 80%，但根据近期趋势预计在所选预测跨度内达到预警值。

80% 为第一版默认值，作为分析配置常量，不修改法定阈值。

### 10.4 风险评分

设备风险分数归一化为 0 至 100，默认组成：

- 当前或范围内最高状态严重度：40%。
- 相对预警值或控制标准的越线幅度：25%。
- 异常持续时间：15%。
- 近期上升趋势：10%。
- 历史异常重复频率：10%。

房间风险由房间内设备风险聚合，兼顾最高设备风险、预警/事故设备比例和异常持续性。报告必须同时输出分数和主要风险原因，避免只给出不可解释的排名。

## 11. 简单预测

### 11.1 预测对象

第一版同时预测：

1. 选中设备、监测类型和单位的未来监测值及越线风险。
2. 系统或筛选范围内未来预警设备数和事故级设备数。

### 11.2 预测跨度

- 下一条记录。
- 未来 1 天。
- 未来 7 天。
- 未来 30 天。

### 11.3 方法选择

候选方法保持可解释：

- 最近值。
- 简单移动平均。
- 线性趋势。
- 指数平滑。

规则：

- 少于 3 个有效点：使用最近值，可信度低。
- 3 至 5 个有效点：比较最近值、移动平均和线性趋势的简单回测误差。
- 6 个及以上有效点：在候选方法中使用时间顺序留出验证，选择平均绝对误差较低的方法。
- 不规则采样使用真实时间间隔拟合，不把记录序号当作等间隔时间。
- 无法形成有效预测时明确显示“数据不足”，不输出伪精确数值。

### 11.4 设备预测状态

预测值与最近一条有效阈值比较，输出：

- 预测值。
- 与预警值、控制标准的距离。
- 预计状态。
- 预计越线时间；只有趋势与数据量支持时才显示。
- 方法、历史样本数、训练时间范围和可信度。

### 11.5 整体风险数量预测

对每台设备各监测类型进行预测后：

1. 按最严重预计状态汇总到设备。
2. 统计预计正常、预警和事故级设备数。
3. 对历史不足或无数据设备单列数量，不将其默认为正常。

所有预测页面和导出报告必须包含“仅供趋势研判，不替代法定监测与事故处置”的说明。

## 12. Excel 生成器

### 12.1 正式录入模板

生成器提供：

- 仅包含表头的通用空白模板。
- 选择监测时间和 1 至多个监测类型后，为 200 台设备预填行的半预填模板。
- 房间 ID、设备 ID 和可选名称预填。
- 监测值留空。
- 单位、预警值和控制标准可按监测类型统一预填，也允许下载后逐行调整。
- 使用数据验证、冻结表头、筛选和基础格式提高录入可用性。

### 12.2 模拟历史数据

可配置：

- 开始和结束时间。
- 采样频率。
- 输出为单个长期总表或多份日表。
- 监测类型、单位、预警值和控制标准。
- 正常、预警、事故设备比例。
- 异常持续时间。
- 快速上升场景。
- 随机种子，保证测试数据可复现。

模拟数据必须遵守正式输入字段规范，并覆盖 20 个房间和 200 台设备。

## 13. 多工作表分析报告

导出文件至少包含：

| 工作表 | 内容 |
|---|---|
| 分析摘要 | 数据范围、覆盖率、状态总数、主要风险结论 |
| 异常事件 | 设备、类型、事件等级、起止时间、持续时间、峰值 |
| 设备风险排名 | 分数、状态、越线幅度、趋势、持续性和原因 |
| 房间风险排名 | 房间分数、预警/事故设备数和主要风险设备 |
| 趋势预测 | 设备预测、预计状态、整体风险数量、方法和可信度 |
| 清洗后的监测数据 | 去重和冲突处理后的标准化记录 |
| 数据质量问题 | 错误、警告、冲突版本和来源文件 |

导出报告使用冻结表头、自动筛选、合理列宽、日期与数值格式，并对状态使用一致颜色。

## 14. 旧系统迁移

### 14.1 移除或替换

- 移除 TXT、DOCX、PDF 报告解析入口，替换为 Excel。
- 移除地面点位、手套孔、障碍物、待测区域和 30 x 28 网格概念。
- 移除旧的污染点位 ID 解析和行列映射。
- 将旧的“污染点位数量预测”替换为监测值和风险设备数量预测。
- 将旧 DOCX 报告生成器替换为 Excel 模板与模拟数据生成器。

### 14.2 可复用部分

- Streamlit 页面组织和视觉组件。
- Plotly 交互选择与详情联动思路。
- 历史回放、趋势、排名和导出交互模式。
- 可解释分析配置面板的设计方式。

旧 DOCX 示例和生成脚本不再参与运行时逻辑；实施阶段可移动至 `legacy/` 或删除，但不得与新 Excel 生成器混用。

## 15. 测试与验收

### 15.1 单元测试

- 房间和设备 ID 校验。
- 设备与房间归属校验。
- 等于预警值和等于控制标准的边界判定。
- 设备和房间最严重状态汇总。
- 日期与日期时间归一化。
- 完全重复去重和冲突选择。
- 单位变化拆分。
- 异常事件合并。
- 风险评分。
- 预测方法选择和低样本回退。

### 15.2 Excel 测试

- 单个长期总表。
- 多文件批量合并。
- 多个候选工作表。
- 缺少必填列。
- 非法 ID、非法时间、非数值监测值。
- 阈值关系错误。
- 阈值随时间变化。
- 单位变化。
- 日期和日期时间混合。
- 生成模板重新导入。
- 模拟数据重新导入。
- 多工作表分析报告结构与内容。

### 15.3 布局和界面测试

- 20 个房间、200 台设备全部显示且 ID 唯一。
- A、B、C 三种布局符合 V5 设计。
- 入口朝向正确，设备、通道和标签不交叠。
- 状态颜色、选中高亮和无数据状态可区分。
- 房间、设备、类型、状态和时间筛选联动。
- 设备详情、趋势和预测与选中对象一致。

### 15.4 端到端验收流程

1. 使用生成器生成含正常、预警、事故和快速上升场景的历史 Excel。
2. 以单总表方式上传并完成分析。
3. 将同一数据按日拆分后批量上传，结果与总表一致。
4. 注入重复和冲突记录，确认质量摘要和默认选择规则。
5. 核对布局图中的设备状态与原始阈值。
6. 核对趋势、异常事件、风险排名和预测解释。
7. 导出多工作表分析报告并重新打开验证。

## 16. 成功标准

设计实现完成后，应满足：

- 用户可通过 Excel 导入长期或批量监测数据。
- 所有 200 台设备可在 V5 空间布局中正确定位和着色。
- 三级状态与阈值边界准确。
- 数据错误、重复和冲突可见且可追溯。
- 用户可查看设备趋势、异常事件、风险排名和简单预测。
- 用户可生成正式录入模板、模拟数据和多工作表分析报告。
- 核心业务逻辑不再集中于单一大型 `app.py`，并具有自动化测试覆盖。
